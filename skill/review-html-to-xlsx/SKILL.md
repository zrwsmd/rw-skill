---
name: review-html-to-xlsx
description: 当用户说“根据代码审查报告_赵瑞文_*.html 生成对应的xlsx文档/汇总表”时，读取该HTML并生成三列xlsx（问题/问题状态/修改计划），状态只允许：严重/中等/轻微/误报，计划只允许：立即修改/本周/本月/不修改，并按严重→中等→轻微→误报排序输出。
allowed-tools: Bash, Read, Glob, Write
---

## 目标输出
- 生成一个 xlsx：三列表头为「问题」「问题状态」「修改计划」。
- 问题状态仅允许：严重 / 中等 / 轻微 / 误报。
- 修改计划仅允许：立即修改 / 本周 / 本月 / 不修改。
- 输出排序：严重 → 中等 → 轻微 → 误报（从上到下）。

## 触发语句识别
当用户的指令包含：
- “根据 代码审查报告_赵瑞文_XXXX.html 生成 xlsx”
- “把 代码审查报告_赵瑞文_XXXX.html 总结成表格/xlsx”
都执行本技能。

## 执行步骤（必须照做）
1) 从用户指令中提取 HTML 文件名（例如：代码审查报告_赵瑞文_20260121084443.html）。
2) 若该文件不存在，则用 glob 查找：代码审查报告_赵瑞文_*.html；若多个，选择最新修改时间的一个，并告知用户选了哪个文件。
3) 运行脚本生成 xlsx（输出默认同名 .xlsx）：
   - python scripts/code_review_report_to_xlsx.py --input "<HTML路径>" --output "<同名xlsx或用户指定>"
4) 生成后在回复里写明输出文件路径（例如：代码审查报告_赵瑞文_20260121084443.xlsx）。
5) 不要把 xlsx 内容贴出来，只需确认生成成功，并提示用户去项目目录查看文件。
