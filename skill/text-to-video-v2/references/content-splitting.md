# 内容拆分指南

将文字内容拆分为多个视频镜头的方法论。

## 核心原则

### 1. 递进式叙述，不重复信息

❌ **错误示例**（重复信息）：
```
镜头1: "苹果公司发布新款 iPhone"
镜头2: "苹果公司推出 iPhone 15"
镜头3: "苹果发布 iPhone 15 系列"
```

✅ **正确示例**（递进叙述）：
```
镜头1: "苹果公司今天举行新品发布会"
镜头2: "推出搭载 A17 芯片的 iPhone 15 Pro"
镜头3: "起售价 999 美元，9月15日开售"
```

### 2. 每个镜头 3-5 秒

- 太短（1-2秒）：观众来不及阅读
- 合适（3-5秒）：阅读舒适，节奏流畅
- 太长（6秒以上）：画面单调，容易失去注意力

### 3. 一个镜头一个信息点

每个镜头只传达一个核心信息，不要塞太多内容。

## 拆分策略

### 策略1：时间线拆分（适用于事件类）

按时间顺序拆分：

```
原文："2026年7月1日，香港举行升旗仪式庆祝回归29周年，飞行队编队飞越维港。"

拆分：
1. "香港特区政府今日举行升旗仪式"（当下）
2. "庆祝香港回归祖国29周年"（背景）
3. "飞行服务队直升机编队飞越维港"（细节）
```

### 策略2：总-分-细拆分（适用于解释类）

从概括到具体：

```
原文："Remotion 是一个用 React 渲染视频的框架，支持逐帧控制和动画效果。"

拆分：
1. "Remotion 是一个视频渲染框架"（总）
2. "使用 React 代码生成视频"（分）
3. "支持逐帧控制和复杂动画"（细）
```

### 策略3：场景拆分（适用于故事类）

按场景/地点拆分：

```
原文："用户打开应用，看到推荐列表，点击视频开始播放。"

拆分：
1. 图片：手机主屏 | "打开应用进入首页"
2. 图片：推荐列表 | "浏览个性化推荐内容"
3. 图片：视频播放 | "点击即可开始观看"
```

### 策略4：要素拆分（适用于产品介绍）

按产品特性拆分：

```
原文："新款耳机支持降噪、续航30小时、防水设计。"

拆分：
1. 图片：耳机特写 | "全新主动降噪耳机"
2. 图片：电池图标 | "长达30小时续航时间"
3. 图片：防水测试 | "IPX7 级防水设计"
```

## 常见错误

### ❌ 错误1：字幕太长

```
"苹果公司今天在加州库比蒂诺总部举行了盛大的新品发布会，正式推出搭载全新 A17 仿生芯片的 iPhone 15 Pro 系列手机"
```

太长了！观众读不完。应该拆成 2-3 个镜头。

### ❌ 错误2：信息重复

```
镜头1: "委内瑞拉发生地震"
镜头2: "委内瑞拉遭遇强震"
```

同一个信息说了两遍，浪费时长。

### ❌ 错误3：跳跃太大

```
镜头1: "小明今天去超市"
镜头2: "他决定创业开公司"
```

中间缺少过渡，逻辑跳跃。

## 拆分检查清单

在完成拆分后，检查：

- [ ] 每个字幕是否递进，没有重复？
- [ ] 每个镜头时长是否在 3-5 秒？
- [ ] 字幕长度是否适中（一般不超过25个汉字）？
- [ ] 镜头之间逻辑是否连贯？
- [ ] 总时长是否符合预期？
- [ ] **镜头数量是不是内容本身决定的，而不是为了凑一个数字硬拆/硬并？**（判断优先级见 `SKILL.md`"镜头数量与视频时长怎么定"一节）

## 时长建议

| 内容类型 | 单条时长 | 镜头数量 | 总时长 |
|---------|---------|---------|--------|
| 短新闻 | 10-15秒 | 2-3个 | 30-60秒 |
| 教程片段 | 15-20秒 | 3-5个 | 1-2分钟 |
| 产品介绍 | 10-15秒 | 2-4个 | 30-60秒 |
| 故事讲述 | 20-30秒 | 4-6个 | 2-3分钟 |
