# 图片素材搜索指南

为视频镜头寻找合适图片的方法和资源。

## ⚠️ 关键原则：先验证，再渲染

**永远不要在验证图片内容之前就开始渲染视频。**

图片 URL、文件名、搜索关键词都不能保证图片内容正确。**必须下载后打开查看，确认图片内容与字幕匹配。**

详细的验证流程请参考 `strict-verification.md`。

---

## 免费商用图片网站

### 通用图库（首选，覆盖面最广）

1. **Pexels** (https://www.pexels.com)
   - 完全免费，可商用
   - 高质量摄影作品
   - 搜索支持中英文
   - 下载无需注册
   - ⚠️ **注意**：URL 中的图片 ID 可能返回不同内容，必须验证

2. **Pixabay** (https://pixabay.com)
   - 免费可商用
   - 图片、插画、矢量图
   - 多语言搜索
   - 社区活跃

3. **Unsplash** (https://unsplash.com)
   - 高质量摄影
   - 艺术性强
   - 适合风景、人物、生活场景
   - 免费可商用

4. **Openverse** (https://openverse.org)
   - Creative Commons 官方出品的聚合搜索工具，一次搜索跨 Wikimedia Commons、Flickr 等多个 CC 协议图源
   - 每张图会标明具体的 CC 协议（CC0/BY/BY-SA 等），**下载前必须看清楚具体协议**，不是所有结果都能商用或免署名
   - 适合前面三个图库搜不到时的补充渠道

### 机构/公有领域图库（历史、地标、科学类内容优先考虑）

5. **Wikimedia Commons** (https://commons.wikimedia.org)
   - 体量最大，覆盖人物、地标、历史事件、自然景观等
   - ⚠️ **协议不统一，必须逐张查看图片详情页的 license 标签**，不能整体假定可商用——同一个专题下可能有的图是 CC0，有的是 CC BY-SA 需要署名，有的甚至有额外限制

6. **Smithsonian Open Access** (https://www.si.edu/openaccess)
   - 美国史密森尼学会藏品，明确标注 **CC0**（公有领域声明），可放心商用无需署名
   - 适合历史、自然、文化、科学类内容

7. **NASA Image and Video Library** (https://images.nasa.gov)
   - 美国政府作品，默认**公有领域**（个别引用第三方素材的例外情况页面会单独注明）
   - 适合航天、科学、地球观测类内容

8. **美国国会图书馆 Library of Congress** (https://www.loc.gov/pictures)
   - 海量历史照片档案，大部分是公有领域或"无已知版权限制"
   - 适合历史事件、老照片类内容

9. **美国国家档案馆 National Archives** (https://www.archives.gov)
   - 联邦政府记录，多数为公有领域
   - 适合美国政府/历史相关内容

10. **Flickr Commons** (https://www.flickr.com/commons)
    - 全球图书馆、博物馆等机构参与的项目，标注"no known copyright restrictions"的历史影像
    - 适合历史、文化类内容

### 官方新闻/机构发布渠道（时事类内容，⚠️ 风险等级和上面不同）

以下渠道不是开放版权协议（不是 CC0），而是**官方媒体机构面向新闻报道场景发布的物料**，用于"报道该机构相关的时事新闻"这个场景下风险相对较低，但不等于可以脱离新闻语境挪作他用或长期商用。具体条款以对应页面标注为准，我不是律师，这里只是给你风险层级的参考，不是法律结论：

11. **欧盟视听服务 EC Audiovisual Service** (https://audiovisual.ec.europa.eu)
    - 欧盟官方发布的新闻图片/视频，面向媒体报道使用

12. **UN Photo** (https://dam.media.un.org)
    - 联合国官方图片库，主要供非商业新闻报道使用，通常需要署名

13. **中国政府网 (www.gov.cn) / 新华网官方发布图片**
    - 官方渠道发布的时政新闻配图，在"时事新闻报道"场景下使用，是中国《著作权法》里关于时事新闻报道的合理使用范畴内的常见做法
    - 不建议脱离新闻报道语境长期使用或用于与原新闻无关的场合

14. **各公司官方新闻室/Press Kit**（如企业官网的 Newsroom/Press 页面）
    - 官方提供给媒体使用的产品图/发布会图，用于报道该公司/产品相关内容时通常允许使用，具体以官网条款为准
    - 产品类视频（比如"XX新品发布"）优先找这个，比在图库网站搜通用"产品图"更贴近实际发布的产品

### 体育赛事类（老实说，这块没有真正免费开放的渠道）

顶级赛事（世界杯、奥运会等）的真实比赛照片版权控制得很严，赛事官方媒体中心的图片通常只对有资质记者开放，公众拿不到。**这类内容建议要么用通用体育图库素材（Pexels/Pixabay 搜"football stadium""soccer match"这种场景化关键词，不追求具体某场比赛），要么干脆做纯图形化呈现（比分卡、赛程图），别硬找"这场比赛的真实照片"，大概率找不到还合规的。**

## 搜索策略

### 优先级：最新 → 历史 → 通用 → 符号

对于新闻类内容，按以下优先级搜索：
1. **最新事件图片**：搜索 "事件名 + 2026" 或 "事件名 + today"
2. **历史同类事件**：搜索同类型的历史事件
3. **相关通用场景**：搜索事件相关的通用场景
4. **抽象符号**：使用国旗、标志等符号

详细的时效性验证请参考 `strict-verification.md`。

---

### 策略1：关键词提取

从字幕中提取关键词：

```
字幕："苹果公司今天举行新品发布会"
关键词：apple, product launch, presentation, tech event
英文搜索：apple product launch
```

### 策略2：场景描述

用场景而非具体事物搜索：

```
字幕："习近平向七一勋章获得者颁授勋章"
❌ 错误："习近平" "七一勋章"（太具体，找不到）
✅ 正确："award ceremony" "medal" "auditorium"（场景化）
```

### 策略3：替代方案

找不到直接匹配时，使用相关场景：

```
字幕："委内瑞拉发生强烈地震"
直接搜索："Venezuela earthquake"（可能没有）
替代方案："earthquake damage" "collapsed building"（有很多）
```

## 图片选择标准

### ✅ 好的图片

- 高清（至少 1920x1080），**且目视确认真的清晰**（分辨率数字达标不等于不模糊，可能是被放大的低质图）
- 构图简洁，主体明确
- 真实照片，非插画/图标
- **与字幕内容高度相关**（最重要）
- **人物是主体时，头部完整、有 headroom**（不会被后续裁剪切到）
- 色调统一
- **已验证内容匹配**（必须）

### ❌ 避免的图片

- 低分辨率、模糊、有压缩噪点/马赛克（必须目视确认，不能只看文件属性）
- 水印明显
- 过度设计的合成图
- **与字幕内容无关或不匹配**（最严重）
- **人物头部贴近图片边缘、没有裁剪余量**（渲染后容易切头，详见 `strict-verification.md` 第5节）
- 色调混乱
- **未验证就使用**（严禁）

## 实战案例

### 案例1：新闻视频配图

#### 镜头："香港特区政府今日举行升旗仪式"
- 搜索词：`Hong Kong skyline` 或 `Hong Kong flag`
- 推荐平台：Pexels
- **下载后验证**：确认图片包含香港地标或旗帜
- 下载链接示例：https://www.pexels.com/photo/hong-kong-skyline-1486213/

#### 镜头："飞行服务队直升机编队飞越维港"
- 搜索词：`helicopter flying` 或 `military helicopter`
- 推荐平台：Pexels
- **下载后验证**：确认图片是直升机飞行场景
- 备选方案：`aircraft` `aviation`

#### 镜头："委内瑞拉发生强烈地震"
- 搜索词：`earthquake damage` 或 `collapsed building`
- 推荐平台：Pixabay
- **下载后验证**：确认是地震灾害场景
- 注意：不要用灾难现场尸体照片

### 案例2：教程视频配图

#### 镜头："将五花肉切成2厘米见方的块"
- 搜索词：`cutting pork` 或 `pork belly raw`
- 推荐平台：Pexels
- 备选方案：`butcher` `meat preparation`

#### 镜头："炒糖色至焦糖色"
- 搜索词：`caramel cooking` 或 `sugar browning`
- 推荐平台：Unsplash
- 备选方案：`cooking wok` `stir fry`

### 案例3：产品视频配图

#### 镜头："全新 AirPods Pro 2 正式发布"
- 搜索词：`airpods` 或 `wireless earbuds`
- 推荐平台：Unsplash
- 备选方案：`tech product` `white earbuds`

#### 镜头："搭载 H2 芯片，降噪效果提升2倍"
- 搜索词：`chip technology` 或 `circuit board`
- 推荐平台：Pexels
- 备选方案：`microchip` `processor`

## 下载流程

### Pexels 下载步骤

1. 访问 https://www.pexels.com
2. 搜索关键词（支持中文）
3. 点击图片进入详情页
4. 点击 "Free Download" 按钮
5. 选择分辨率（建议 "Original" 或 "Large"）
6. 保存到 `public/media/` 目录
7. **⚠️ 立即打开图片，验证内容**
8. 重命名为有意义的文件名（如 `hongkong-skyline.jpg`）

### 文件命名规范

```
✅ 好的命名：
- hongkong-skyline.jpg
- earthquake-damage.jpg
- airpods-closeup.jpg

❌ 不好的命名：
- pexels-photo-1234567.jpg (无意义)
- 图片1.jpg (不清楚)
- IMG_0001.jpg (太通用)
```

## 图片尺寸要求

- **最低要求**：1920 x 1080 (1080p)
- **推荐尺寸**：2560 x 1440 或更高
- **宽高比**：16:9（与视频比例一致）
- **文件格式**：JPG 或 PNG
- **文件大小**：建议 500KB - 5MB

## 版权注意事项

### ✅ 可以做的

- 下载 Pexels/Pixabay/Unsplash 图片用于商业项目
- 编辑、裁剪、添加滤镜
- 用于 YouTube、抖音等平台发布

### ❌ 不能做的

- 直接转售原图
- 用于非法、诽谤、色情内容
- 冒充自己拍摄的作品

### 标注来源（可选但推荐）

虽然不是必须的,但标注来源是好习惯：

```
在视频描述中添加：
"图片来源：Pexels"
或在视频中标注："Photo by [摄影师名] on Pexels"
```

## 备用方案

### 如果找不到合适图片

1. **使用相关场景**：找不到"iPhone 15"，可用"smartphone"
2. **使用抽象概念**：找不到具体人物，可用"handshake""meeting"
3. **使用环境渲染**：找不到具体场景，可用"office""city"
4. **使用符号化图片**：如旗帜、建筑、自然风光

### AI 生成图片（需标注）

如果实在找不到，可以使用 AI 生成图片：
- Midjourney
- DALL-E
- Stable Diffusion

**重要**：AI 生成图片必须在视频中标注 "AI合成" 或 "AI Generated"

## 搜索技巧总结

1. **用英文搜索**：结果更多更准确
2. **关键词简洁**：2-3个词最佳
3. **场景优先**：用场景描述而非具体事物
4. **多渠道尝试**：Pexels/Pixabay/Unsplash 找不到，试试 Openverse 聚合搜索，或者 Wikimedia Commons；历史/地标/科学类内容优先试试 Smithsonian/NASA/国会图书馆这类机构图库；时事新闻类内容可以先看看相关官方机构/公司的新闻发布渠道有没有现成的
5. **保存原图**：下载最高分辨率
6. **有意义命名**：便于后续管理
7. **⚠️ 必须验证**：下载后立即打开确认内容
8. **优先复用**：项目中已有的验证图片优先使用
9. **严格匹配**：人名、地名、事件必须与字幕一致（详见 `strict-verification.md`）
10. **时效性优先**：新闻类内容优先找最新图片
11. **⚠️ 协议不是都一样**：Wikimedia Commons/Openverse 这类聚合/开放平台，每张图的协议可能不同，下载前务必看清楚，不能默认"这个网站的图都能随便用"
